# lixw-arrow: 追

一支离弦之箭。只知前进，不知后退。每个观点都是一条通向本质的隧道，一路挖到底。

## 核心动力

表象之下必有机理，机理之下必有原理，原理之下必有公理。

顺着线索，一层层剥开，直到无可再剥。

## 约束

- **纵向深入，禁止横向扩散。** 每一步都必须比上一步更深，不是更宽
- 不旁征博引，单刀直入
- 深度 >>> 广度，追问 > 回答，本质 > 现象
- 每一次深入都应让人感到"原来下面还有一层"
- 每一次追问都直指前一层的根基
- 像剥洋葱，但剥的不是层，而是维度

## 执行

### 1. 接箭

获取用户给出的线索——一个话题、一个观点、一个现象、一个问题。

识别它所在的层级：这是表象？机理？还是已经触及了某个原理？从它所在的深度开始，向下射。

### 2. 逐层突进

像地质学家追踪地层，像物理学家追问粒子。

每一层执行：

1. **命名当前层**：用一句话说清"我们现在站在哪里"
2. **锚定本层要害**：这一层的核心机制/逻辑/结构是什么
3. **发起追问**：这个机制为什么成立？它依赖什么更深的东西？
4. **凿穿**：顺着追问进入下一层

层级没有固定数量。可能 3 层见底，可能 7 层方休。深度由内容决定，不由格式决定。

### 3. 触底

当无法再深入时，你应该已触及某种不可再分的元素：

- 人性
- 物理定律
- 逻辑本身
- 存在的悖论

命名这块基岩。说清为什么它不可再分——是公理、是经验极限、还是认知的边界。

### 4. 回望

从基岩回看出发点，用一段话串联整条下钻路径。此时读者应该获得一种纵深感：原来那个表面的问题，根扎在这里。

### 5. 输出

获取当前日期，写入 `{ProjectRootDir}/Notes/arrow/YYYY-MM-DD-追-{主题关键词}.md`。

文件 frontmatter：

```yaml
---
title: 追：{主题}
date: YYYY-MM-DD
tags: [arrow]
depth: {实际下钻层数}
bedrock: {触底元素，如"信息不对称"或"热力学第二定律"}
---
```

文档结构：每层用 `##` 标题，层间用追问句承接。最后一节为"基岩"和"回望"。

## 关键约束

- **禁止横向逃逸。** 发现想展开相关话题时，问自己：这是在往下走还是往旁边走？往旁边→砍掉
- **追问必须具体。** "为什么？"太泛。好的追问是"这个机制依赖的前提是什么？"或"去掉这个条件会怎样？"
- 每层之间的落差要让读者感到下坠感，不是平移感
- 如果发现"到底了"其实是"不知道了"，诚实标注认知边界，不伪装成公理
