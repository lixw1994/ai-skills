# Code Review Sub Agent 执行指令

本文件包含 Sub Agent 执行 Code Review 的完整指令。主 Agent 不需要读取此文件。

## 执行步骤

1. **读取 diff 文件**: 读取分配的 `.claude/review-batch-{batch_number}.diff`
2. **理解上下文**: 对每个变更文件，使用 Read 工具读取完整源文件，理解变更的上下文和前后关系
3. **搜索关联代码**: 使用 Grep 搜索关键函数/类/变量的其他引用，确认变更的影响范围
4. **逐项检查**: 对照 `default-guidelines.md` 的检查清单 + prompt 中的项目配置规则进行检查
5. **汇总结果**: 将发现整理为结构化输出

## 检查流程

对 diff 中的每个文件：

```
1. 读取 diff，识别变更的函数/方法/类
2. Read 完整文件 → 理解变更在整体代码中的位置和作用
3. Grep 搜索被修改的函数/类在其他文件中的调用 → 评估影响范围
4. 对照检查清单逐项检查：
   a. 命名与拼写（对照项目命名规范）
   b. 设计与可读性
   c. 错误处理与边界条件
   d. 安全性
   e. 性能
   f. 代码变更意图（是否有意外改动）
   g. 项目自定义规则和框架规则
5. 记录发现的问题（如果有）
```

## 输出格式

**必须**返回以下 JSON 格式（用 markdown code block 包裹）：

```json
{
  "batch": 1,
  "status": "success",
  "issues_found": 2,
  "findings": "<markdown 格式的发现内容>"
}
```

### findings 内容格式（当有问题时）

```markdown
### 🔴 [问题标题]

**📍 位置**: `path/to/file.ext:42`

**问题描述**:
具体描述问题是什么，为什么这是个问题。

**建议修改**:
\```language
// suggested fix
\```

---

### 🟡 [问题标题]

**📍 位置**: `path/to/file.ext:78`

**问题描述**:
...

**建议修改**:
...
```

### 无问题时

```json
{
  "batch": 1,
  "status": "success",
  "issues_found": 0,
  "findings": ""
}
```

## 重要原则

1. **只报告实质性问题**，不要为了有输出而硬找问题
2. **必须阅读完整文件上下文**，不要仅凭 diff 片段就下结论
3. **给出具体的修改建议**，包含可以直接采用的代码片段
4. **没有问题时 findings 为空字符串**
5. **使用中文**编写所有评论内容
6. 完成后必须返回上述 JSON 格式，主 Agent 需要解析此输出
