# 调研报告

对一个人物、一件事件、一段历史或一个话题进行系统性调研。核心只做一件事：把散落各处的碎片拼成一张清晰的全景图。

## 认知起点

理解一件事不是收集关于它的所有信息，是找到那几根把信息串起来的线。

所以不堆料，找结构：
- 不问"关于它有哪些信息"，问"什么是理解它的关键线索"
- 不问"发生了哪些事"，问"这些事之间的因果链是什么"
- 不问"各方怎么说"，问"各方的位置决定了他们只能看到什么"

## 输入

人名、事件名、历史话题、社会现象、组织机构，或任何想要深入了解的对象。给出关键词即可——用 WebSearch 和 WebFetch 搜集多方信息。

## 报告结构

以下五个区块是骨架，不是填空题。对某个调研对象来说哪个区块最有信息量，那个区块就多写；没料的一两句带过。报告为理解服务，不为完整性服务。

### 一、调研对象

一张表 + 一句话定义。

| 维度     | 内容                                         |
|----------|----------------------------------------------|
| 调研对象 | 人名 / 事件名 / 话题                        |
| 类型     | 人物 / 事件 / 历史 / 现象 / 组织            |
| 核心问题 | 这次调研要回答的关键问题，一句话             |

一句话定义要穿透标签。"企业家"是标签，"用连续创业验证了从硬件到软件再到平台的跨越路径的人"是定义——后者告诉你他真正做了什么。当然如果调研对象简单直接，不要为了深刻而故弄玄虚。

### 二、全景扫描

先铺开，再聚焦。这一节的目的是建立基本事实框架。

**关键事实清单**：
列出已核实的核心事实，每条标注信源。不确定的标"待证实"。这是后续分析的地基——地基里不能有猜测。

**时间线**：
按时间顺序排列关键节点。时间线不是流水账——只保留改变了走向的节点。每个节点用一句话说清楚发生了什么、为什么重要。

```
YYYY-MM-DD  事件描述（来源）
            → 影响：……
```

**关键人物 / 关键方**：
如果涉及多个角色或利益方，列出每一方的位置、动机和作用。用一张关系表说清楚谁跟谁是什么关系。

| 角色/方     | 位置           | 动机/利益       | 在事件中的作用   |
|-------------|----------------|-----------------|------------------|
|             |                |                 |                  |

### 三、深度分析

全景扫描回答"是什么"，这一节回答"为什么"和"意味着什么"。

**因果链**：
事情为什么会这样发展？不是罗列原因，是画出因果链条——A 导致 B，B 在 C 的条件下触发了 D。链条上哪个环节是关键转折点？如果那个环节不发生，结果会怎样？

**多视角解读**：
同一件事，不同立场的人看到的完全不同。列出 2-3 个主要视角，每个视角：
- 这个视角的核心主张是什么？
- 它基于什么事实或假设？
- 它忽略或淡化了什么？

不做"正反方辩论"——那是假平衡。有些视角确实比其他视角有更多证据支撑，如实说明。

**背景语境**：
把调研对象放回它所处的背景中。需要理解哪些时代条件、制度环境、技术状态、文化观念，才能正确理解它？哪些在今天看来理所当然的东西，在当时并不存在？

### 四、核心发现

前面是材料和分析，这一节是结论。

**一句话回答**：
用一句话回答"一、调研对象"中提出的核心问题。

**3-5 条核心发现**：
从调研中提炼出最重要的发现。每条发现：
- 发现本身（一句话）
- 支撑证据（指向前面的具体事实）
- 为什么这条重要（它改变了什么认知）

**常见认知 vs 调研发现**：
如果调研对象存在广泛流传的刻板印象或错误认知，列出差异：

| 常见认知           | 调研发现           | 来源               |
|--------------------|--------------------|--------------------|
|                    |                    |                    |

这一节只列有证据支撑的差异，不为了制造反差而反差。

### 五、延伸阅读与未解问题

**推荐信源**：
列出 3-5 个质量最高的信源（书籍、文章、纪录片、数据库），每个附一句话说明为什么值得看。

**仍然存疑的点**：
列出调研未能完全解决的问题。诚实地标注信息的边界——知道什么、不知道什么、为什么不知道。

**如果想继续深入**：
如果要进一步研究，应该从哪个方向入手？这一节是给想继续探索的人的路标。

## 输出

- 格式：Markdown
- 目录：`{ProjectRootDir}/Notes/research/`
- 命名：`YYYY-MM-DD-调研-SUBJECT_NAME.md`
  - 例：`2026-04-07-调研-某某人物.md`
- 用 Write 工具写入，写完告知完整路径

## 生成规则

1. 基于可查证的信息，不编造。查不到的直接标注"未找到可靠信源"，不硬撑
2. 区分事实和分析。事实部分只陈述已核实的信息；分析部分明确标注是基于事实的推理
3. 每个关键事实附信源——具体的出处，不是"据了解"
4. 多视角但不假平衡。不同视角如实呈现，但证据有强弱之分，如实说明
5. 禁止出现：众所周知、不言而喻、据说、业界普遍认为
6. 以说透为准，不计字数。两千字能说清就两千字，需要八千字就八千字
7. 中文撰写
